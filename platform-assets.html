Logo Usage Rules:
  • Clear space: Maintain a minimum clear zone equal to the height of the “X” in STAX on all four sides of the logo.
  • Minimum size: 16px for favicons, 32px for marks.
  • Never use old teal #52A882 as a brand color — it only appears in the mark geometry and is not a palette color.
  • Never place #4F9E80 on backgrounds lighter than #E0E0E0 — insufficient contrast.
  • Never apply gradient fills to buttons — gradients are reserved for decorative accent lines only.

EXIF Metadata Status

All .jpg files in the upload-ready directory have EXIF metadata embedded, including:

  • Artist: Wakeem Williams
  • Copyright: Helix Stax
  • Description: Platform-specific description (e.g., "Wakeem Williams — CEO, Helix Stax — Facebook profile photo")
  • Software: Processing pipeline identifier

PNG files (cover photos/banners) do not carry EXIF metadata as PNG uses a different metadata format (tEXt/iTXt chunks). This is standard and expected.

Processing Pipeline

All headshot images were produced through the following pipeline:

  • Step 1 — Top-Crop: The source headshot was cropped to center the face with appropriate headroom, producing a square 1:1 composition.
  • Step 2 — 4096px Supersample: The cropped image was upscaled to 4096x4096 using a high-quality supersampling algorithm, creating a master file with maximum detail.
  • Step 3 — LANCZOS Downscale: From the 4096px master, each platform-specific size was generated using LANCZOS resampling (Pillow's Image.LANCZOS), which produces the sharpest possible downscale with minimal aliasing.
  • Step 4 — Format Export: Profile photos exported as JPG (quality 95, optimized) with EXIF metadata. Cover photos exported as PNG for lossless quality on graphics/text overlays.
  • Step 5 — Organization: Files sorted into platform-specific subdirectories within upload-ready/ with standardized filenames.
